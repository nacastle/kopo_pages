HW02

[ 소스트리(Sourcetree) 기초 ]

  1. 소스트리 개념
    2. 개발자들에게는 자신이 개발한 소스 관리가 늘 큰 부담이 됩니다. 그래서 이것을 관리하고자 다양한 형상관리 툴을 쓰고 있지요. 대표적인 
툴이라고 한다면 SVN, CVS 등이 있을 것입니다. 요즘에는 우리나라에서 git을 많이 쓰고 있지요. git을 사용하는 방법은 여러 가지가 있습니다.
          
"소스트리는 환경을 시각화해서 쉽게 git을 사용할 수 있게 도와주는 프로그램입니다."
IDE 툴인 이클립스에서 git을 연결해서 사용할 수도 있죠. 그리고 git bash 등을 사용해 CLI환경에서 직접 명령어를 쳐서 git을 사용할 수도 있습니다.그러
나 GUI 환경에 익숙한 분들의 경우에는 CLI가 매우 낯설기 마련이죠. 바로 이 과정을 좀 더 시각화하여 쉽게 git을 사용할 수 있게 도와주는 프로그램이 
'소스트리'입니다.
  2. 소스트리 사용영상
    3. 다음 아래의 작업을 수행해 보았습니다.


    - Clone
- Branch 생성, 체크아웃, 삭제
- Commit
- Pull


HW01

[ 깃허브 기초 ]

  1. 저장소(Repository)
    2. 프로젝트가 거주(live)할 수 있는 디렉토리나 저장 공간. 깃허브 사용자는 종종 “repo”로 줄여서 사용한다. 당신의 컴퓨터 안의 로컬 폴더가
될 수도 있고, 깃허브나 다른 온라인 호스트의 저장 공간이 될 수도 있다. 저장소 안에 코드 화일, 텍스트 화일, 이미지 파일을 저장하고, 이름붙일 수 있다.
  2. 브랜치(Branch)
    3. 여러 명이 하나의 프로젝트에서 깃 없이 작업하는 것이 얼마나 혼란스러울 것인가? 일반적으로, 작업자들은 메인 프로젝트의 브랜치를 따와서
(branch off), 자신이 변경하고 싶은 자신만의 버전을 만든다. 작업을 끝낸 후, 프로젝트의 메인 디렉토리인 “master”에 브랜치를 다시 “Merge”한다.
  3. 커밋(Commit)
    4. 깃에게 파워를 주는 명령이다. 커밋하면, 그 시점의 당신의 저장소의 “스냅샷”을 찍어, 프로젝트를 이전의 어떠한 상태로든 재평가하거나 
복원할 수 있는 체크포인트를 가질 수 있다.
  4. 깃허브 사용영상
    5. 다음 아래의 작업을 수행해 보았습니다.


    - Repository 생성/삭제
- Branch 생성/삭제
- File Upload
- Commit
- Read-me
